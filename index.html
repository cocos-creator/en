What is Fireball?

Fireball is an editor framework for game development. We focus on creating the most accessible and feature reach GUI toolset in one single editor with HTML and JavaScript. On the runtime side, Fireball uses other popular open sourced game engines such as Pixi.js and Cocos2D-JS. This makes Fireball really easy to get into for game engine users. Users will keep the benefit of all the features, packing and optimization of other game engines, but with a better editor much more can be achieved.

The editor design is deeply inspired by Unity which we believe has the most advanced game developing editor at the moment. Fireball advances in editor and tool extendability thanks to modern Web technology. We use a combination of Polymer, Electron, Nodejs and plain-old HTML and CSS. Web developers can easily hack the editor and create tools that empowers your workflow in no time.

Fireball Features

Integrated Cross-platform Editor

Using a framework powered by Polymer and Electron, Fireball has integrated full-featured toolset in a single cross-platform desktop app. Developers can complete a full workflow containing tasks such as scene building, animation, particle, bitmap font, run your game and deploy to mobile platform all in the same editor.

Supports Multiple Game Engines

Probably the first game development editor that are plugged with different open sourced game engines. Start with Pixi.js and Cocos2D-JS, we will be supporting engines written in C++ in the future. Make it extremely flexible for users to choose engines that best suit their needs, while keeping the same GUI toolset to create content-rich games.

Extend the Editor with Packages

Fireball has a very slim core structure. Beyond that are all plugins (or packages). Build a package to extend the editor with new panels is as easy as building a HTML page with JavaScript functions. Users can choose packages for a certain project to keep things minimal. Also user can import any npm or bower packages into their projects, all good with JavaScript.

Play as You Build

We make sure developers can hit the "Play" button any time in the editor to see what's going on with the game. When playing in editor users can still tweak things around like component properties and object positions. With Chrome's DevTools you can even change your scripting while playing and see the difference immediately.

Customizable Data Management and Inspecting

From assets to components, user can control the serialization and deserialization for every bit of data defined. With a bit of customization, data can be shown in Inspector just the way you want.

Automatic Build, Preview and Deploy

Powered with gulp.js, Fireball can take over build pipeline with a single button. Once a project is built, you can immediately access your game from mobile browser. Continuous integration is also another click away, thanks to fully JavaScript workflow.

Fireball In Action

Assets and Data Inspecting

Inspecting and editing all kinds of assets and data, each has its own type of properties. You can write your own data interpreter to display complex data structures.

Debugging With DevTools

Using Chromium as the core, you can debug and profiling anything running in Fireball with powerful DevTools. Editing scripts during runtime is also possible!

One Button Build and Preview

Besides play in editor, build your project for mobile devices is just one button away. Fireball will setup web server automatically so that you can access your game directly on mobile devices.
